    **## What is Mesh?**
    Brief intro paragraph: Mesh is a statically-typed, compiled programming language designed for expressive, readable concurrency. It combines the actor model from Erlang/Elixir with a modern type system, pattern matching, and native compilation via Rust. Mention key properties: statically typed with inference, compiles to native code, actor-based concurrency with supervision, familiar syntax inspired by Elixir and Rust.

    **## Installation**
    ### Prerequisites
    - Rust toolchain (for compilation): `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh`
    ### Building from Source
    ```bash
    git clone https://github.com/user/mesh.git
    cd mesh
    cargo build --release
    ```
    ### Verifying Installation
    ```bash
    ./target/release/mesh --version
    ```

    **## Hello World**
    Create a file `hello.mpl`:
    ```mesh
    fn main() do
      println("Hello, World!")
    end
    ```
    Compile and run:
    ```bash
    mesh hello.mpl
    ```
    Explain: `fn main()` is the entry point. `do...end` blocks define function bodies. `println` prints to stdout with a newline.

    **## Your First Program**
    A slightly more complex example showing variables, string interpolation, and functions:
    ```mesh
    fn greet(name :: String) -> String do
      "Hello, ${name}!"
    end

    fn main() do
      let message = greet("Mesh")
      println(message)
    end
    ```
    Explain: `let` for variable binding, `::` for type annotations, `->` for return types, string interpolation with `${}`, last expression is the return value.

    **## What's Next?**
    Links to Language Basics (/docs/language-basics/) for the full language tour, Type System (/docs/type-system/) for advanced types, and Concurrency (/docs/concurrency/) for actors and message passing.

    CRITICAL: All code examples MUST use ```mesh fences for syntax highlighting. Base all Mesh syntax on the e2e test files -- do NOT invent syntax. Use `fn main() do...end` pattern, `let` for variables, `::` for type annotations, `->` for return types.

    **# Language Basics** (h1 for page title)

    **## Variables**
    - `let` bindings: `let x = 42`, `let name = "Mesh"`
    - Type annotations: `let x :: Int = 42`
    - Variables are immutable by default
    - Show examples with Int, Float, String, Bool types

    **## Basic Types**
    - Int, Float, String, Bool, Nil
    - String interpolation: `"Hello, ${name}!"`
    - Type inference: the compiler infers types from context
    - Show a table or list of built-in types

    **## Functions**
    - `fn name(params) do...end` syntax
    - Type annotations: `fn add(a :: Int, b :: Int) -> Int do...end`
    - Last expression is the return value (no explicit return keyword)
    - Multi-line functions
    - Show 2-3 examples of increasing complexity

    **## Pattern Matching**
    - `case expr do...end` syntax
    - Matching on values: `0 -> "zero"`, `1 -> "one"`, `_ -> "other"`
    - Matching on types/constructors (brief intro, detailed in Type System docs)
    - Destructuring in case arms
    - Show example from research: fn describe(x :: Int) -> String do case x do 0 -> "zero" ...

    **## Control Flow**
    - `if/else`: `if condition do...else...end`
    - `case` expressions (covered in Pattern Matching)
    - `for` loops: `for item in collection do...end`
    - `while` loops: `while condition do...end`
    - `break` and `continue`
    - Show examples for each construct

    **## Pipe Operator**
    - `|>` pipes the left-hand value as the first argument to the right-hand function
    - `5 |> double |> add_one` is equivalent to `add_one(double(5))`
    - Chaining multiple transformations
    - Show the double/add_one example from research

    **## Error Handling**
    - `try/catch` blocks
    - Result types and pattern matching on results
    - Show a basic try/catch example based on e2e test patterns

    **## Modules**
    - `module Name do...end` syntax
    - Module functions accessed with dot notation: `Math.add(1, 2)`
    - Show a simple module definition and usage
